### Task 1：修复 agent_name 未注入 configurable

**根本原因：** `frontend/src/core/threads/hooks.ts:215-219` 只把 `agent_name` 写入 thread metadata（用于 UI 路由），但 `thread.submit()` 的 `context`（= LangGraph configurable）里没有它。后端 `make_lead_agent` 读 `cfg.get("agent_name")`，永远拿到 `None`，agent 始终以 default 模式运行。

**Files:**
- Modify: `frontend/src/core/threads/hooks.ts:490-506`
- Create: `frontend/tests/unit/core/threads/agent-name-context.test.ts`

- [x] **Step 1: 理解当前上下文结构**

打开 `frontend/src/core/threads/types.ts`，确认 `AgentThreadContext` 已有 `agent_name?: string` 字段（L19）。这意味着 `context.agent_name` 类型正确，不需要修改类型定义。

- [x] **Step 2: 写失败测试**

新建 `frontend/tests/unit/core/threads/agent-name-context.test.ts`：

```typescript
import { describe, it, expect } from "vitest";

// 验证 AgentThreadContext 包含 agent_name 字段的类型约束
import type { AgentThreadContext } from "@/core/threads/types";

describe("AgentThreadContext", () => {
  it("includes agent_name as optional string field", () => {
    const ctx: AgentThreadContext = {
      thread_id: "test-thread",
      model_name: undefined,
      thinking_enabled: false,
      is_plan_mode: false,
      subagent_enabled: false,

    };
    expect(ctx.agent_name).toBe("my-agent");
  });

  it("allows agent_name to be undefined", () => {
    const ctx: AgentThreadContext = {
      thread_id: "test-thread",
      model_name: undefined,
      thinking_enabled: false,
      is_plan_mode: false,
      subagent_enabled: false,
    };
    expect(ctx.agent_name).toBeUndefined();
  });
});
```

- [x] **Step 4: 修改 hooks.ts，将 agent_name 注入 context**

打开 `frontend/src/core/threads/hooks.ts`，找到 `thread.submit()` 调用（约 L463），修改 `context` 对象：

```typescript
// 修改前（L490-506）：
context: {
  ...extraContext,
  ...context,
  thinking_enabled: context.mode !== "flash",
  is_plan_mode: context.mode === "pro" || context.mode === "ultra",
  subagent_enabled: context.mode === "ultra",
  reasoning_effort:
    context.reasoning_effort ??
    (context.mode === "ultra"
      ? "high"
      : context.mode === "pro"
        ? "medium"
        : context.mode === "thinking"
          ? "low"
          : undefined),
  thread_id: threadId,
},

// 修改后：
context: {
  ...extraContext,
  ...context,
  agent_name: context.agent_name,
  thinking_enabled: context.mode !== "flash",
  is_plan_mode: context.mode === "pro" || context.mode === "ultra",
  subagent_enabled: context.mode === "ultra",
  reasoning_effort:
    context.reasoning_effort ??
    (context.mode === "ultra"
      ? "high"
      : context.mode === "pro"
        ? "medium"
        : context.mode === "thinking"
          ? "low"
          : undefined),
  thread_id: threadId,
},
```

注意：`...context` 已经展开了整个 context 对象（含 `agent_name`），但为了明确语义和避免被 `extraContext` 覆盖，显式写出 `agent_name: context.agent_name`。

### Task 2：将默认语言改为中文

**Files:**
- Modify: `frontend/src/core/i18n/locale.ts:3`

- [x] **Step 1: 确认影响范围**

`detectLocale()` 在 `locale.ts:44-54` 读取 `navigator.language`，英文浏览器用户不受影响。`normalizeLocale()` 在 L27-41 对中文前缀 (`zh*`) 自动映射到 `zh-CN`。只有「首次访问且浏览器语言未知/不支持」时才会 fallback 到 `DEFAULT_LOCALE`。

- [x] **Step 2: 修改 DEFAULT_LOCALE**

编辑 `frontend/src/core/i18n/locale.ts` 第 3 行：

```typescript
// 修改前
export const DEFAULT_LOCALE: Locale = "en-US";

// 修改后
export const DEFAULT_LOCALE: Locale = "zh-CN";
```

预期：无错误，无测试失败（`zh-CN` 是 `SUPPORTED_LOCALES` 中的合法值）
