# M6: Audit (Middleware + Batch Writer + Query + Export) Implementation Plan

> **For agentic workers:** Detail level: **task-list level**. Expand to TDD per step when executing.

**Goal:** Land the full audit pipeline — Gateway middleware, async batch writer, fallback file log, query API, CSV export, retention job. Wire the stubs left in M3/M5.

**Prerequisites:** M5 merged. Branch `feat/m6-audit`.

**Spec reference:** §9 (Audit).

**Non-goals:** SIEM adapters; full-text search; charts; anomaly detection; before/after diff.

**Flag-off invariant:** The `AuditMiddleware`, `AuditBatchWriter`, retention job, and audit router are only registered when `ENABLE_IDENTITY=true`. With the flag off, no PG connection is attempted and no background task is spawned. The M1 regression guard `tests/identity/test_feature_flag_offline.py` is extended here to additionally assert `/api/audit/*` returns 404 when flag is off.

## Task 2: AuditBatchWriter

```python
class AuditBatchWriter:
    def __init__(self, session_maker, *, queue_max: int = 10000, flush_interval_sec: float = 1.0, batch_size: int = 500): ...
    async def start(self) -> None: ...
    async def stop(self) -> None: ...   # drain up to 5s
    async def enqueue(self, event: AuditEvent, *, critical: bool = False) -> None:
        """If queue full and critical → sync write. Otherwise drop + metric."""
    async def _flush_loop(self) -> None: ...
```

Implementation notes:
- On PG failure: call `fallback.write(event)` for critical events; non-critical dropped + metric `audit_write_failures_total`
- Write-coalescing: within 10s window, same (action, user_id, resource_id) tuple occurring >100 times → collapse to one row with `metadata.count`

Tests (`test_writer.py`):
- enqueue N events → all appear in DB after flush
- queue_max reached + critical → sync write succeeds
- queue_max reached + non-critical → event dropped, metric increments
- PG offline → fallback path writes JSONL + retries on next flush when reconnected
- stop() drains within 5s

## Task 3: Gateway AuditMiddleware

```python
class AuditMiddleware(BaseHTTPMiddleware):
    def __init__(self, app, writer: AuditBatchWriter, *, audited_reads: Sequence[str] = AUDITED_READS): ...
    async def dispatch(self, request, call_next):
        t0 = time.perf_counter()
        response = await call_next(request)
        dt = int((time.perf_counter() - t0) * 1000)
        if self._should_audit(request, response):
            event = self._build_event(request, response, dt)
            await self.writer.enqueue(event, critical=self._is_critical(event.action))
        return response
```

Helpers: `should_audit`, `_build_event`, `derive_action`, `extract_resource_type`, `extract_resource_id`.

`AUDITED_READS = {"/api/auth/oidc/", "/api/audit", "/api/admin/audit"}` (prefix match).

Register before IdentityMiddleware so audit sees identity from request.state (set by IdentityMiddleware downstream)? Actually **before** = outermost; AuditMiddleware sets `t0` then calls next, which hits IdentityMiddleware which populates `request.state.identity`, which AuditMiddleware reads after downstream returns. Confirm ordering with a unit test.

Tests (`test_middleware.py`):
- POST request → event enqueued with correct action, result, duration
- GET /api/me → NOT enqueued (excluded)
- GET /api/audit/* → enqueued
- 5xx response → result="failure" + error_code from header
- Path params extracted for resource_id

## Task 5: Query API

```python
@router.get("/api/tenants/{tid}/audit", dependencies=[Depends(requires("audit:read", scope="tenant"))])
async def list_audit(
    tid: int,
    user_id: int | None = None,
    action: str | None = None,
    resource_type: str | None = None,
    result: str | None = None,
    date_from: datetime | None = None,
    date_to: datetime | None = None,
    cursor: str | None = None,
    limit: int = 50,
): ...

@router.get("/api/tenants/{tid}/audit/export", dependencies=[Depends(requires("audit:read", scope="tenant"))])
async def export_audit_csv(tid: int, ... filters): ...
```

Cursor encoding: `base64(f"{created_at_iso}|{id}")`. Default time window 7 days, max 90 days — enforce 400 if exceeded.

Export: `StreamingResponse` streaming CSV; hard cap 100k rows → 413; emits `audit.exported` event with metadata of filters.

Platform-admin sees cross-tenant via `/api/admin/audit` (requires `audit:read.all`).

Tests:
- Filters combine correctly
- Cursor pagination stable across writes
- 100k+1 rows → 413
- Platform admin sees all tenants
- Non-admin tenant_owner sees only their tenant
- export itself is logged
